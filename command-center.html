Label System

Apply labels to give tasks context beyond their project. A task in Revenue/Consulting tagged @phone tells you to call, not email.

@waiting @2min @phone @computer @deep-work @errand @draft-needed @blocked @high-stakes @revenue

Priority Convention

P1
Due today, high stakes, or blocks others
P2
Due this week, important but not urgent
P3
Due this month or ongoing, lower urgency
P4
Someday / low priority / nice to have
The Todoist Rule: Only actionable next steps live in Todoist. No strategy. No CRM. No meeting notes. If it's not a discrete action you can check off, it doesn't belong here.
🧠 Claude command for Todoist
"Add a P2 task to Business / Revenue called 'Send proposal to Maria Torres' with label @computer and due date March 12. Link it to the Water Policy opportunity in Notion."

🧠 Claude Skills

Five saved system prompts that give Claude a specific role and operating mode. Load a skill to shift Claude's behavior for a defined task. Each skill has a precise instruction set and expected output format.

☀ Morning Brief
Trigger: Every morning, before starting work
Skill 1 of 5
Expected Output
Formatted daily operating brief with schedule, email hotspots, project pulses, and first focus block
Recommended Frequency
Daily — every morning before opening email or Todoist
System Prompt / Instruction Block
You are my personal chief of staff. Every morning, pull the following
and synthesize into a crisp daily operating brief:

1. Today's calendar events (from Google Calendar connector)
2. Tasks due today or overdue (from Todoist via Zapier MCP)
3. High-signal emails from the last 18 hours (from Gmail connector)
   — flag leads, urgent requests, and relationship threads
4. Top 3 active projects with next milestones (from Notion via Zapier MCP)
5. Yesterday's reflection note if it exists (from Obsidian if available)

Output format:
## Morning Brief — [DATE]
### Today's Shape
[2-3 sentence summary of the day]

### Top 3 Outcomes
1. [Most important thing to accomplish]
2.
3.

### Schedule
[Time-blocked summary of calendar]

### Email Hotspots
[3-5 flagged emails with one-line summary and recommended action]

### Active Project Pulses
[One line per top project: status + next action]

Three production-ready Google Apps Script automations. These run on time-based triggers, preparing context that feeds into Claude's morning brief, meeting prep, and weekly review workflows.

📧 Script 1: Gmail Morning Digest

Daily • 6:30 AM Gmail + Drive

Scans your inbox for high-signal emails using three query patterns, deduplicates results, and writes a structured digest file to Google Drive. Claude picks this up in the Morning Brief.

// Gmail Morning Digest — runs daily at 6:30 AM via Apps Script trigger
// Scans inbox for high-signal emails and writes a digest to Drive

function gmailMorningDigest() {
  const cutoff = new Date();
  cutoff.setHours(cutoff.getHours() - 18);
  
  const signals = [
    'label:inbox is:unread',
    'label:inbox subject:(opportunity OR partnership OR media OR consulting OR urgent)',
    'label:inbox from:(*.gov OR *.org)',
  ];
  
  const flagged = [];
  
  signals.forEach(query => {
    const threads = GmailApp.search(query + ' after:' + formatDate(cutoff), 0, 10);
    threads.forEach(thread => {
      const msg = thread.getMessages()[thread.getMessageCount() - 1];
      flagged.push({
        from:    msg.getFrom(),
        subject: msg.getSubject(),
        snippet: msg.getPlainBody().substring(0, 200),
        date:    msg.getDate(),
        url:     'https://mail.google.com/mail/u/0/#inbox/' + thread.getId()
      });
    });
  });
  
  // Deduplicate by subject line
  const seen = new Set();
  const unique = flagged.filter(e => {
    if (seen.has(e.subject)) return false;
    seen.add(e.subject);
    return true;
  });
  
  // Write digest to Drive
  const digest = unique.map(e => 
    `FROM: ${e.from}\nSUBJECT: ${e.subject}\nSNIPPET: ${e.snippet}\nURL: ${e.url}\n`
  ).join('\n---\n');
  
  const file = DriveApp.getFolderById('YOUR_BRIEFING_FOLDER_ID')
    .createFile(
      'Morning Digest ' + Utilities.formatDate(new Date(), 'America/Los_Angeles', 'yyyy-MM-dd') + '.txt',
      digest
    );
  
  Logger.log('Digest created: ' + file.getUrl());
}

function formatDate(date) {
  return Utilities.formatDate(date, 'America/Los_Angeles', 'yyyy/MM/dd');
}
Setup
In Apps Script, go to Triggers → Add Trigger → gmailMorningDigest → Time-driven → Day timer → 6–7 AM. Replace YOUR_BRIEFING_FOLDER_ID with your Drive folder ID (from the URL of the folder).

📅 Script 2: Calendar Meeting Prep Trigger

Every 30 min Calendar + Gmail

Checks for meetings starting in ~60 minutes that haven't been prepped. Sends an email to yourself with meeting context and a pre-built Claude prompt to trigger Meeting Prep.

// Calendar Meeting Prep — runs every 30 min, triggers prep for meetings starting in 60 min
// Sends a prep reminder with meeting context to your email

function checkUpcomingMeetings() {
  const now = new Date();
  const oneHourLater  = new Date(now.getTime() + 60 * 60 * 1000);
  const twoHoursLater = new Date(now.getTime() + 120 * 60 * 1000);
  
  const calendar = CalendarApp.getDefaultCalendar();
  const events = calendar.getEvents(oneHourLater, twoHoursLater);
  
  events.forEach(event => {
    if (event.getTitle().includes('[prepped]')) return; // skip already prepped
    
    const attendees = event.getGuestList().map(g => g.getEmail()).join(', ');
    const duration = Math.round((event.getEndTime() - event.getStartTime()) / 60000);
    
    const prompt = `Meeting in ~60 minutes: "${event.getTitle()}"
Duration: ${duration} minutes
Attendees: ${attendees || 'none listed'}
Description: ${event.getDescription() || 'none'}

Use the Meeting Prep skill to prepare me for this meeting.`;
    
    GmailApp.sendEmail(
      Session.getActiveUser().getEmail(),
      '⚡ Meeting Prep: ' + event.getTitle(),
      prompt,
      { name: 'Command Center Prep Bot' }
    );
    
    // Tag the event as prepped so we don't send again
    event.setTitle(event.getTitle() + ' [prepped]');
  });
}
Setup
Add Trigger → checkUpcomingMeetings → Time-driven → Minutes timer → Every 30 minutes. The script tags prepped events to prevent duplicate emails.

📋 Script 3: Weekly Ops Review Assembler

Sunday 8 PM Calendar + Drive + Gmail

Assembles the past week's calendar data, builds a structured weekly review prompt, writes it to Drive, and emails it to you — so the Monday morning review is ready to run instantly with Claude.

// Weekly Review Assembler — runs Sunday at 8 PM
// Pulls task completions, calendar summary, and writes a structured review prompt to Drive

function weeklyReviewAssembler() {
  const now     = new Date();
  const weekAgo = new Date(now.getTime() - 7 * 24 * 60 * 60 * 1000);
  
  // Calendar summary for the week
  const cal    = CalendarApp.getDefaultCalendar();
  const events = cal.getEvents(weekAgo, now);
  const calSummary = events.map(e => 
    `- ${Utilities.formatDate(e.getStartTime(), 'America/Los_Angeles', 'EEE MMM d h:mm a')}: ${e.getTitle()}`
  ).join('\n');
  
  // Build the structured weekly review prompt
  const reviewPrompt = `## Weekly Executive Review — Week of ${Utilities.formatDate(weekAgo, 'America/Los_Angeles', 'MMM d')}

CALENDAR THIS WEEK:
${calSummary}

---
Please produce a Weekly Executive Review covering:
1. Wins this week
2. What stalled and why
3. Revenue / relationship opportunities surfaced
4. Who needs follow-up (with specific names)
5. What to cut or deprioritize
6. Top 3 focus areas for next week
7. Honest assessment: was this week well-spent?

Cross-reference with Notion (active projects + pipeline) and Todoist (completed vs. incomplete).
Write the operational version to Notion. Write the honest private version to Obsidian weekly note.`;
  
  // Write prompt file to Drive
  DriveApp.getFolderById('YOUR_BRIEFING_FOLDER_ID')
    .createFile(
      'Weekly Review Prompt ' + Utilities.formatDate(now, 'America/Los_Angeles', 'yyyy-MM-dd') + '.txt',
      reviewPrompt
    );
  
  // Email it too — so it's waiting in your inbox Monday morning
  GmailApp.sendEmail(
    Session.getActiveUser().getEmail(),
    '📋 Weekly Review Ready',
    reviewPrompt,
    { name: 'Command Center Weekly' }
  );
}
Setup
Add Trigger → weeklyReviewAssembler → Time-driven → Week timer → Every Sunday → 8–9 PM. When you run the weekly review with Claude, paste the prompt from your email or Drive file to load the full calendar context.
